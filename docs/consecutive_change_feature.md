# 连续涨跌统计功能说明

## 功能概述

在股票分析结果中新增了两个统计指标：
1. **连续涨跌日** - 记录股票连续上涨或下跌的天数
2. **连续涨跌幅度** - 记录这段连续涨跌的总幅度

## 显示规则

### 连续涨跌日
- **正数**：表示连续上涨天数，如 `+3天` 表示连续上涨3天
- **负数**：表示连续下跌天数，如 `-2天` 表示连续下跌2天
- **0天**：表示平盘或单日波动

### 连续涨跌幅度
- **正数**：表示累计上涨幅度，如 `+5.23%` 表示这几天累计上涨了5.23%
- **负数**：表示累计下跌幅度，如 `-3.45%` 表示这几天累计下跌了3.45%

## 计算逻辑

系统从最新交易日往前回溯，找到价格变化方向相同的连续天数：

```
示例1：连续上涨
日期      收盘价    涨跌
10-23    10.00     --
10-24    10.20     ↑  ← 开始
10-25    10.50     ↑
10-26    10.80     ↑  ← 当前
结果：连续涨跌日 = +3天
      连续涨跌幅度 = +8.00% (从10.00涨到10.80)

示例2：连续下跌
日期      收盘价    涨跌
10-23    20.00     --
10-24    19.70     ↓  ← 开始
10-25    19.40     ↓  ← 当前
结果：连续涨跌日 = -2天
      连续涨跌幅度 = -3.00% (从20.00跌到19.40)
```

## 输出位置

### 1. 控制台输出
在详细分析结果表格中会显示这两列：

```
股票名称  代码        建议  信心度   当前价格    涨跌幅     连续日    连续幅度    风险  决策理由
平安银行  000001.SZ  买入  75.00%   11.39元    -0.52%    -4天      -2.06%     中等  技术面显示...
```

### 2. CSV文件
在 `outputs/<timestamp>/analysis_summary.csv` 中会包含这两列数据

### 3. JSON文件
在 `outputs/<timestamp>/analysis_detailed.json` 中会包含这两个字段：
```json
{
  "股票代码": "000001.SZ",
  "股票名称": "平安银行",
  "连续涨跌日": "-4天",
  "连续涨跌幅度": "-2.06%",
  ...
}
```

## 实现文件

1. **计算工具** - `src/utils/consecutive_change_calculator.py`
   - `calculate_consecutive_changes()` - 核心计算函数
   - `format_consecutive_days()` - 格式化天数显示
   - `format_consecutive_change()` - 格式化幅度显示

2. **集成位置** - `main.py`
   - 在 `_analyze_stock_with_cache()` 方法中计算统计数据
   - 在 `print_analysis_results()` 方法中显示结果

3. **测试脚本** - `test/test_consecutive_changes.py`
   - 包含4个测试场景：连续上涨、连续下跌、混合涨跌、真实数据
   - 运行方式：`python test/test_consecutive_changes.py`

## 使用示例

运行主程序后，系统会自动计算并显示这些统计：

```bash
python main.py
```

输出示例：
```
[完成] 平安银行(000001.SZ) 分析完成:
    操作: 持有 | 信心度: 65.00% | 价格: 11.39元 | 涨跌: -0.52%
    连续涨跌日: -4天 | 连续涨跌幅度: -2.06%
```

## 注意事项

1. 该统计基于历史收盘价计算
2. 需要至少2天的历史数据才能计算
3. 如果数据不足或异常，会返回默认值（0天，0.00%）
4. 连续计算会在价格方向改变时中断并重新开始计数
