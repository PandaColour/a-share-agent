# 雪球登录态问题解决方案

## 🐛 问题描述

您的洞察完全正确：
> "我觉得问题在于雪球需要一个登录态"

雪球网站在没有登录的状态下：
- ✅ 可以访问首页
- ✅ 可以查看股票基本信息
- ❌ **无法访问讨论区内容**

这导致网页抓取器被误认为是技术问题（选择器失效、反爬虫拦截等），实际上是**登录态问题**。

## 🔍 根本原因分析

### 雪球的访问策略

1. **公开数据**（无需登录）：
   - 股票基本信息
   - 历史价格数据
   - 公司基本资料

2. **用户生成内容**（需要登录）：
   - 用户讨论帖子
   - 评论和回复
   - 用户观点分享

### 我们的抓取目标

情感分析需要的是**用户讨论内容**，这正好是雪球的**登录保护区域**。

## ✅ 解决方案

### 1. 配置禁用雪球抓取

**文件**: `config/unified_config.json`

```json
"web_scraping": {
  "enabled": true,
  "description": "网页数据抓取配置（Playwright增强）- 雪球需要登录已禁用",
  "eastmoney_news": true,
  "xueqiu_discussions": false,
  "reason_xueqiu_disabled": "雪球讨论区需要登录态，未登录无法获取内容"
}
```

### 2. 代码逻辑适配

**文件**: `src/data/web_scraper.py`

```python
# 检查配置中的雪球启用状态
from config.config_manager import get_config
config = get_config()
web_config = config.get('analysis_settings.web_scraping', {})
xueqiu_enabled = web_config.get('xueqiu_discussions', True)

if xueqiu_enabled:
    logger.info("雪球讨论已启用，开始抓取...")
    tasks.append(scraper.scrape_xueqiu_discussions(symbol, max_items=15))
else:
    logger.info("雪球讨论已禁用（需要登录态），跳过抓取")
    async def empty_discussions():
        return []
    tasks.append(empty_discussions())
```

### 3. 系统行为变更

**修复前**：
```
尝试抓取雪球讨论 → 需要登录 → 被拦截 → 抓取失败 → 错误日志
```

**修复后**：
```
检查配置 → 雪球已禁用 → 跳过抓取 → 使用东方财富新闻 → 正常运行
```
